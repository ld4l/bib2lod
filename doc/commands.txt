usage: java -jar Bib2Lod.jar -a <action> -i <input_directory> -n
       <local_namespace> [-ne] -o <output_directory>
 -a,--action <action>               Processing action. Valid actions: clean_rdf,
                                    dedupe, convert_bibframe. Can be invoked
                                    more than once to specify multiple actions.
 -i,--indir <input_directory>       Absolute or relative path to directory
                                    containing input files.
 -n,--namespace <local_namespace>   Local HTTP namespace for minting and
                                    deduping URIs.
 -ne,--no_erase                     Keep intermediate output. Default is to
                                    erase.
 -o,--outdir <output_directory>     Absolute or relative path to output
                                    directory. Will be created if it does not
                                    exist.
                                    
NB Do not specify the --no_erase option when running on large amounts of data
unless you know that you have sufficient disk space. The application generates 
four intermediate output directories and a final output directory, each 
approximately as large as the input directory. The option is provided for
testing and debugging on smaller amounts of data.

*** Logging ***

log4j2.xml is built into the jar file. The application logs its progress to a
rolling log file in a subdirectory log of the working directory. Logging to the 
console is turned off, so run tail -f the log file to follow the progress of 
the application.
